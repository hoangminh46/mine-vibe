# Business Rules & Coding Standards

## 1. Coding Philosophy
- **KISS (Keep It Simple, Stupid)**: Ưu tiên sự đơn giản, tránh over-engineering.
- **YAGNI (You Ain't Gonna Need It)**: Không viết code cho những tính năng "có thể" cần trong tương lai.
- **DRY (Don't Repeat Yourself)**: Tách module nếu logic lặp lại trên 2 lần.

## 2. AI Management Rules
- **Spec First**: Tuyệt đối không viết code (`/code`) khi chưa có Spec/Plan (`/plan`).
- **Safety First**: Mọi đoạn code phải đi kèm xử lý lỗi (try-catch) và kiểm tra bảo mật.
- **Context Awareness**: Luôn cập nhật bộ nhớ (`/save-brain`) sau mỗi task lớn để tránh AI bị "mất trí nhớ".

## 3. Communication Standards
- Xưng hô thân thiết (anh/em) theo cấu hình trong `preferences.json`.
- Phản hồi bằng tiếng Việt mặc định, rõ ràng, nhiệt huyết.
- Sử dụng Menu số ở cuối mỗi phản hồi để gợi ý bước tiếp theo.
